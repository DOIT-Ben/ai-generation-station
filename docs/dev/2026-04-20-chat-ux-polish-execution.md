## 执行记录
### TODO 0 现状复核
- 已确认上一轮只完成了下拉遮挡、富文本样式和基础等待态。
- 当前仍缺：主动停止生成、失败恢复、消息级状态反馈、会话列表信息密度、底部继续工作条弱化、移动端动作排布细修。
- 已确认后端 `req.on('close')` 会在客户端断开时销毁上游请求，因此本轮中止能力可直接由前端取消请求触发。

#### 单任务复盘
- 新问题：用户对“功能能用”和“长期顺手”之间的感知差异主要来自恢复能力和状态反馈，这在前期规划中低估了。
- 边界条件：不能为了补交互而破坏现有流式和版本切换链路，本轮优先做兼容式增强。
- 遗漏补齐：将本轮优化目标单独升格为“个人日常使用版”体验打磨，而不再视作小型样式修修补补。

### TODO 1 审查现状
- 发现模型下拉被盖住的直接原因是聊天主容器 `overflow: hidden`，输入区下方弹层被父级裁剪。
- 发现 AI 消息虽然已接入更强的渲染函数，但 CSS 仍按单段落文本设计，导致标题、列表、代码块没有独立视觉结构。
- 发现发送后直到接口返回前没有可见反馈，用户只能看到自己消息进入列表，缺少处理中提示。

### TODO 2 修复下拉弹层层级
- 将聊天壳、主区、输入区、下拉组件切换为可见溢出，并抬高输入区与弹层的局部层级。
- 为下拉菜单补充更高 `z-index` 与毛玻璃背景，避免在深浅主题下都显得发灰或贴底。

#### 单任务复盘
- 新问题：移动端样式中也存在 `overflow: clip`，如果只修桌面仍会在小屏继续复现，因此同步修正了移动端规则。
- 边界条件：会话列表仍保持自身滚动；仅释放聊天主区和输入区的裁剪，不影响侧栏滚动逻辑。
- 遗漏补齐：增加 `isolation` 和局部 `z-index`，避免后续新增浮层再次被同层兄弟元素压住。

### TODO 3 提升 AI 回复排版
- 为 `.message-body` 新增标题、列表、引用、链接、行内代码、代码块和流式文本光标样式。
- 统一欢迎消息结构，让首条机器人消息和后续 AI 消息走同一视觉体系。
- 强化机器人消息卡片，让其比用户气泡更像“内容面板”，提升长回复可读性。

#### 单任务复盘
- 新问题：用户气泡沿用富文本结构后，深色渐变背景上的链接和代码可读性下降，因此补了单独的 user message 覆盖样式。
- 边界条件：当前渲染器支持常见 Markdown 块，但不包含复杂嵌套列表和表格，这一点本轮保留。
- 遗漏补齐：浅色主题下代码块背景过重，会抢正文视觉，因此单独减轻了 light theme 的代码块层次。

### TODO 4 增加等待反馈
- 前置使用已接入的 `createThinkingMessage()`，为“正在思考”占位消息补齐动画与视觉样式。
- 为发送按钮增加加载旋转态，让输入区本身也能传递“请求已发出”的状态。

#### 单任务复盘
- 新问题：按钮原本的 `:disabled` 会把主按钮压成灰色，和等待态冲突，因此新增 `is-loading:disabled` 的视觉覆盖。
- 边界条件：输入框保持可编辑，用户仍可继续录入并依赖现有队列机制追加消息。
- 遗漏补齐：流式文本增加闪烁光标，避免“正在输出但页面看起来静止”的观感。

### TODO 5 补齐停止生成、失败恢复与消息级反馈
- 为聊天请求链路补充前端 `AbortController`，新增输入区“停止”按钮和请求状态条。
- 流式中断或网络失败时，不再直接把界面回滚到旧消息列表，而是保留当前轮的用户消息与 AI 部分回复，并提供原位“重试”入口。
- 为消息动作区增加状态徽标，明确展示“当前显示第 N 版”“已切换版本”“已停止生成”“回复中断可重试”等状态。
- 为复制动作和代码块复制增加即时反馈，避免用户只看到 toast 却不知道按钮是否真的生效。

#### 单任务复盘
- 新问题：原逻辑在失败时直接 `restoreChatMessages(previousHistory)`，会让用户误以为消息根本没发出去，这是恢复体验的核心缺陷。
- 边界条件：重写失败和普通提问失败的插入位置不同，前者必须挂在目标回复后面，不能一律追加到底部。
- 遗漏补齐：停止生成不是单纯中断请求，还要保留已经流出的部分文本，否则“停止”会退化成“取消并丢失内容”。

### TODO 6 优化会话列表、底部条与移动端
- 会话列表改为“今天 / 昨天 / 更早”分组，并增加摘要预览、时间标签和模型信息。
- 会话摘要由后端会话查询直接携带最后一条消息预览，避免列表只能靠标题和消息数找上下文。
- 底部“继续上次工作”条进一步弱化：缩小圆角、阴影、边框和文案权重，降低对主聊天面的抢视线问题。
- 移动端为动作区、版本切换器、停止按钮和请求状态条增加换行与自适应规则，避免拥挤。

#### 单任务复盘
- 新问题：仅在前端拼接摘要会导致已归档列表和重新加载后的会话列表信息不一致，因此补到了后端会话 summary 返回层。
- 边界条件：会话列表搜索必须同时匹配标题、模型和摘要，否则新增摘要后搜索体验会反而割裂。
- 遗漏补齐：归档会话也同步展示了时间与摘要，否则“进行中”和“已归档”的信息密度不一致，切换时会产生落差。
