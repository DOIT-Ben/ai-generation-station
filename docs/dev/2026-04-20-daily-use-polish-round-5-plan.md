# 2026-04-20 Daily Use Polish Round 5 Plan

## 目标
- 继续推进“长期天天用”的最后一轮细磨，重点不再是补功能，而是降低高频操作的认知成本和界面占位。
- 本轮只处理三个高频使用节点：
  - 聊天消息操作反馈：点击复制 / 重写 / 切换版本后，页面要立即表达“系统正在做什么”。
  - 后台审计筛选效率：让管理员快速切到常用日志范围，并清楚知道当前究竟筛了什么。
  - 移动端顶部导航：减少手机端 portal 顶栏占位，避免顶部长期像一块展开工具条。

## 设计方向
- 聊天区：所有消息操作尽量采用“就地反馈”，不要让用户只靠 toast 判断有没有生效。
- 审计区：筛选应像运营台而不是表单，常用条件一键命中，复杂条件可见且可清除。
- 顶部导航：桌面端保持轻量悬浮导航，移动端默认收起，按需展开，不挤压正文空间。

## 顺序化 TODO
1. 审查聊天消息动作、审计日志筛选和 portal 顶栏的当前实现边界，补齐 round-5 文档。
2. 优化聊天消息微交互：为重写 / 版本切换补充按钮级反馈和消息内状态提示。
3. 优化后台审计筛选：增加常用快捷筛选、活跃筛选摘要和一键移除能力。
4. 优化移动端 portal 顶部导航：增加折叠面板和切换按钮，降低顶部占位。
5. 更新执行日志，执行 JS 语法检查与前端烟测，确认通过后提交 git。

## 验收标准
- 用户点击聊天动作后，页面能立刻给出明确的本地反馈，而不是只等异步结果。
- 管理员可以更快筛到“今天 / 最近 7 天 / 密码重置 / 邀请签发”等高频日志范围。
- 手机端 portal 顶栏默认更紧凑，展开后仍可完整访问工作台 / 个人中心 / 后台 / 退出链路。
