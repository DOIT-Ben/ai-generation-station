# 2026-04-21 Daily Use Polish Round 16 Plan

## 目标
- 把聊天页现有的轻量“消息摘录”升级成更适合个人长期使用的摘录管理流。
- 继续坚持同页闭环，不新增独立页面，不把功能塞进顶栏或导航栏。
- 本轮仍只做前端与本地持久化，避免范围膨胀到后端资产系统。

## 设计结论
- 方案选择：在聊天主区内把摘录架升级为“可展开的管理面板”。
- 不选单独页面：会打断聊天连续性，且当前阶段价值不高。
- 不选侧边栏常驻：聊天区已经是主工作面，侧栏继续加模块会显得更挤。

## 顺序化 TODO
1. 复核现有摘录架、聊天输入框与动作事件结构，建立 round-16 文档。
2. 扩展摘录状态，增加面板展开态、视图筛选态与搜索输入支持。
3. 重构摘录架 UI，加入展开管理、筛选、搜索、批量复制能力。
4. 为单条摘录加入再次利用动作：复制内容、插入到输入框继续提问、跳回原消息、移除。
5. 打磨样式与移动端响应，保证折叠态不占空间、展开态可读。
6. 补齐 round-16 文档和静态断言。
7. 执行回归测试，确认无回归后提交推送。

## 预期结果
- 默认状态下：摘录架仍然轻量，只展示必要信息和展开入口。
- 需要整理时：用户可直接在聊天页展开管理、搜索和筛选摘录。
- 需要复用时：摘录内容能一键复制或回填到输入框，形成真正的工作流闭环。
