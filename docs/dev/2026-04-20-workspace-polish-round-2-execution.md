## 执行记录
### TODO 1 审查现状
- 已确认模板区已有模板保存和收藏能力，但卡片层级弱、无法搜索、无法预览模板核心内容。
- 已确认侧栏仍偏宽，导航项和 logo 占用较重，对主内容横向空间侵占明显。
- 已确认聊天区阅读排版已经完成基础结构化，但长段落和列表仍偏“工具输出感”。
- 已确认会话列表已有分组和预览，但缺少行内快捷操作，管理动作仍偏绕路。

#### 单任务复盘
- 新问题：模板卡和会话列表都已经承载更多信息，继续叠内容会让界面更乱，因此必须同步做层级重构。
- 边界条件：不能把本轮改成大规模重写，必须与现有状态逻辑兼容。
- 遗漏补齐：模板区和会话区都需要从“能展示”升级到“能管理”。 

### TODO 2 升级模板区
- 为各功能模板库增加了搜索输入、结果统计、模板预览摘要和内容字数信息。
- 修正了模板筛选后的应用映射逻辑，改为在渲染时直接保留原始 group/item 索引，避免筛选后“应用模板”错位。
- 模板卡视觉改为更像工具箱：标题、描述、预览、状态、主动作分层明确，收藏与应用操作分开。

#### 单任务复盘
- 新问题：模板过滤后如果继续依赖渲染后的按钮顺序回推原始索引，逻辑会非常脆弱。
- 边界条件：模板预览内容可能来自 `message`、`values.prompt`、`values.text` 或 description，必须做统一兜底。
- 遗漏补齐：模板标题和描述渲染时补上了转义，避免非预期 HTML 注入页面。

### TODO 3 瘦身导航栏与主工作区
- 压缩了侧栏宽度、logo 尺寸、导航项高度和底部配额区占用，让主工作区获得更多横向空间。
- 同步调整 `main` 主内容区和 tablet 断点下的侧栏宽度，避免只改桌面宽度导致中屏布局错位。
- 保留原有配额折叠能力，但整体观感从“重卡片”切到更轻的工作台样式。

#### 单任务复盘
- 新问题：侧栏不是单独存在，`main`、tablet 断点和移动端抽屉宽度都与它耦合。
- 边界条件：不能破坏移动端现有抽屉式导航，因此仅压缩桌面和平板宽度，不动移动端抽屉逻辑。
- 遗漏补齐：平板宽度同步从 232 调整到 220，避免主区域仍按旧宽度计算。

### TODO 4 精修聊天阅读排版
- 优化了 AI 回复气泡的宽度控制、内边距、段落节奏、列表间距、引用块和行内代码样式。
- 让机器人回复气泡更像阅读面板，用户消息则保持更紧凑，避免两侧视觉重量一样重。
- 保持现有流式输出、版本切换、复制和重写能力不变，只精修可读性层。

#### 单任务复盘
- 新问题：回复已经支持重写、重试、版本切换，阅读区样式不能只服务纯文本，还要兼容结构化富文本。
- 边界条件：不能把回复区域拉得过宽，否则长段落在桌面端会失去阅读节奏。
- 遗漏补齐：为段落增加了 `text-wrap: pretty`，减少中文长句换行割裂感。

### TODO 5 提升会话列表管理感
- 会话列表改成“主卡片 + 行内动作”的对象管理样式，不再只有单个大按钮。
- 增加了行内 `改名` / `归档` 快捷动作，并补充 `当前` / `空白` 状态 pill。
- 归档逻辑升级为支持按会话 id 处理，避免只能操作当前会话；非当前会话归档时不会打断当前聊天上下文。

#### 单任务复盘
- 新问题：旧结构如果把动作按钮继续塞进会话按钮内部，会形成无效的嵌套按钮结构。
- 边界条件：归档当前会话和归档非当前会话的后续行为不同，必须分开处理。
- 遗漏补齐：补了 `archiveConversationById` / `renameConversationById`，把顶部管理按钮和行内按钮统一到同一套逻辑。

### TODO 6 回归与收尾
- 完成 JS 语法检查、页面结构断言和 UI 烟测。
- 测试过程中发现一个静态断言直接匹配旧的归档确认文案源码片段，不是运行时问题；已通过兼容写法保留原中文确认文案并继续复用新的按 id 归档逻辑。

#### 单任务复盘
- 新问题：测试脚本存在对源码字符串的硬编码断言，重构时即使行为正确也可能误报失败。
- 边界条件：不能为了通过测试回退新的会话归档逻辑，因此选择兼容旧文案片段而不是撤销重构。
- 遗漏补齐：补跑了 `node --check` 与两轮测试，确认修复后没有引入新的回归。
