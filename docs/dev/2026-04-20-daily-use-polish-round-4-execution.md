- 新增 starter 卡片点击后自动填入聊天输入框并聚焦，直接衔接后续发送，不打断现有草稿与队列状态。
- 保留原消息壳和 `message-body` 结构，避免破坏聊天样式和现有页面结构断言。

#### 单任务复盘
- 新问题：首屏引导不能只是静态说明，否则仍然没有降低“第一句要怎么说”的成本。
- 边界条件：不能破坏流式回复和空会话恢复链路，因此必须挂在现有 `restoreChatMessages()` 空态分支上，而不是另起一套页面外壳。
- 遗漏补齐：starter 卡片点击后同步刷新 composer 状态和自动保存，避免只填入输入框、不更新状态条。

### TODO 3 升级后台批量管理
- 在后台用户列表上方新增批量工具条，支持“全选当前筛选结果”“清空选择”“批量启用 / 禁用 / 设为 Pro / 设为 Free”。
- 用户卡片新增多选框，并增加已选数量和当前筛选范围提示，让管理员知道这次批量动作会影响谁。
- 批量动作复用现有 `updateAdminUser` 能力按用户顺序执行，结束后统一刷新用户列表与审计日志。

#### 单任务复盘
- 新问题：管理员高频操作不是“找不找得到按钮”，而是“同一动作要不要重复点很多次”。
- 边界条件：没有新的批量后端接口，前端只能在现有接口之上做顺序编排，因此不能设计成过于激进的大事务 UI。
- 遗漏补齐：批量禁用时明确跳过当前管理员自己，避免误把自己踢下去。
