# 2026-04-20 Daily Use Polish Round 4 Plan

## 目标
- 继续推进“长期天天用”的最后一层体验，不再只做单点美化，而是补高频冷启动和管理员高频维护动作。
- 本轮聚焦两块主路径：
  - 聊天首屏：第一次进入或新建空会话时，不再只有一条欢迎语，而是提供可立即开问的引导面板。
  - 管理后台：从单账号逐个点按钮，升级到带筛选和批量动作的管理台。

## 设计方向
- 聊天首屏：像真正的 AI 工作台首页，给出“我现在就能问什么”的起手式，而不是让用户自己想第一句。
- 管理后台：像运营工具而不是调试台，目标账号应能快速定位，批量动作应能低摩擦完成。
- 响应式：所有新结构都必须同步考虑手机宽度，不允许桌面增强、移动端散架。

## 顺序化 TODO
1. 审查聊天空会话渲染链路、后台单用户操作链路与移动端相关样式边界。
2. 升级聊天首屏：增加欢迎面板、快捷提问卡片和一键填入输入框的 starter prompts。
3. 升级后台用户管理：增加多选、全选当前筛选结果、清空选择和批量状态 / 套餐动作。
4. 补齐新结构的移动端样式与状态反馈。
5. 更新执行文档，执行语法检查与前端烟测，完成提交推送。

## 验收标准
- 空会话进入后，用户无需自己想第一句话，能直接从卡片中选一个方向继续。
- 管理员能先搜索，再批量启用 / 禁用 / 切换套餐，而不是逐卡片重复点操作。
- 新增结构在移动端下仍可正常浏览和操作，不出现按钮墙或布局塌陷。
