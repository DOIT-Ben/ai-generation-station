# 2026-04-20 Chat UX Polish Plan

## 目标
- 修复聊天模型下拉框被遮挡的问题，确保弹层稳定浮出输入区。
- 提升 AI 回复排版，支持标题、列表、引用、链接、行内代码与代码块的可读展示。
- 在用户发送消息后立即提供“正在思考”过渡反馈，降低等待期间的空白感。

## 顺序化 TODO
1. 审查聊天区容器层级、输入区溢出裁剪、消息渲染链路与等待态入口。
2. 调整聊天壳层级和下拉弹层样式，消除输入区附近的裁剪问题。
3. 补齐聊天消息富文本样式，统一首条欢迎消息和流式消息的视觉结构。
4. 为发送按钮与 AI 占位消息增加等待反馈样式，明确“已发送，正在处理”状态。
5. 更新静态断言与开发过程文档，执行前端回归测试并记录结果。

## 验收标准
- 聊天模型下拉展开后不被聊天容器或输入区截断。
- AI 回复不再是一整块裸文本，常见 Markdown 结构具备明确层级。
- 发送消息后在首个接口返回前就能看到可感知的等待态。
