- 已确认模板库已经支持搜索和收藏，但“刚用过的模板再来一次”这类高频动作仍然要重新翻列表。

#### 单任务复盘
- 新问题：当前摩擦集中在“定位”而不是“能力缺失”，尤其是会话和模板一多之后，重复动作的成本开始放大。
- 边界条件：本轮不改接口，不新建服务端状态，所有增强必须落在本地 UI 推导和现有偏好存储上。
- 遗漏补齐：新增的目录、最近使用和检索反馈都必须兼容现有聊天渲染与模板应用链路，不能引入额外状态分叉。

### TODO 2 优化聊天侧栏搜索反馈
- 在会话搜索框下方增加了结果反馈区，默认展示进行中 / 已归档数量，筛选时会直接提示匹配数量。
- 当当前对话被筛选条件排除时，会明确提示“当前对话未命中筛选”，并提供清空筛选和回到当前对话按钮。
- 新增“定位当前对话”动作，让用户不用手动在长会话列表里重新找当前项。

#### 单任务复盘
- 新问题：侧栏搜索的主要成本不是输入关键词，而是筛完以后不知道自己把什么筛掉了。
- 边界条件：不能改变现有会话列表排序和分组逻辑，因此反馈区只能作为附加状态层，而不是重写列表结构。
- 遗漏补齐：回到当前对话时要兼容“当前对话被筛掉”的场景，因此先清空筛选再滚动定位。

### TODO 3 优化长回复阅读
- 在聊天主区增加了“本条回复目录”，当最近一条 AI 回复包含多个标题段落时，会自动生成目录 chips。
- 点击目录项后可直接平滑滚动到对应标题位置，降低长回复里手动来回滚动定位的成本。
- 回复目录依赖现有 markdown 标题结构，和消息正文分离渲染，不影响原有正文样式和复制 / 重写等动作。

#### 单任务复盘
- 新问题：长回复的真正问题不只是“长”，而是读到一半后回跳某个段落的成本太高。
- 边界条件：不能新增服务端结构摘要，因此目录必须完全由现有正文里的 `h1/h2/h3` 自动推导。
- 遗漏补齐：流式回复完成后最终仍会走整段消息重渲染，所以目录锚点需要在静态消息渲染阶段统一补齐。

### TODO 4 优化模板续接
- 在模板库里增加“最近使用”区域，记录各功能最近点过的模板，并支持直接再次应用。
- 最近使用信息写入本地工作台续接状态，登录后仍能沿用，不依赖后端新增接口。
- 当模板搜索没有命中时，最近使用仍然会保留在顶部，方便用户直接回到刚才用过的模板。

#### 单任务复盘
- 新问题：模板库的问题并不是模板少，而是“刚用过的模板下一次还得重新找”。
- 边界条件：不能新增模板服务端字段，因此最近使用只能记录模板在当前库中的位置索引，并在渲染时做有效性校验。
- 遗漏补齐：索引型记录方式可能遇到模板库变化，所以渲染最近使用时必须过滤已经失效的引用。
