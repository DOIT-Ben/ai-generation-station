# 2026-04-21 Daily Use Polish Round 12 Plan

## 目标
- 继续把聊天页打磨到更适合长期高频使用的状态，聚焦消息级操作收纳、移动端键盘与长会话浏览细节、空白态到继续追问的节奏过渡。
- 本轮仍然只处理聊天页前端与其静态测试，不触碰外部脏文件 `public/js/app-shell.js`。

## 设计方向
- 操作层级化：把每条 AI 回复的动作分成“主动作”、“次动作”和“版本导航”，减少横向平铺按钮造成的尾部噪音。
- 移动端视口感知：让输入区、滚动按钮和消息区域在软键盘弹起时更稳定，不出现底部控件被压住或长会话阅读被打断。
- 会话阶段感：区分“未开始对话”、“刚开始第一轮”、“已有上下文继续推进”三种状态，让建议文案和输入区反馈更顺。

## 顺序化 TODO
1. 复核 round-11 基线与当前聊天页结构，建立 round-12 计划、执行、回归文档。
2. 重构消息级操作区：收纳复制、重写、重试、版本切换的层次与状态反馈，减少长回复尾部的视觉负担。

4. 优化空白态与继续追问节奏：让 starter、建议带、请求状态和输入区文案随会话阶段自然切换。

## 验收标准
- AI 回复尾部的操作区不再一排堆满按钮，主次层级清晰，版本切换更容易理解。
- 手机端键盘弹起后，输入区和回到底部按钮位置稳定，长会话浏览时不会频繁丢失阅读点。
- 首次进入聊天、发出第一条消息、准备继续追问三个阶段的文案和建议入口更自然，没有重复提示和节奏断裂。
- 文档、回归记录、提交范围完整可追溯，并继续排除 `public/js/app-shell.js`。
