# 2026-04-20 Template Library Enrichment Plan

## 目标
- 将工作台内所有模板从“短提示词”升级为“可直接产出高质量首轮结果”的长模板。
- 保持现有前端模板结构不变，不新增字段，不改模板入口交互，只增强模板内容本身。
- 覆盖 `chat`、`lyrics`、`music`、`cover`、`speech`、`covervoice` 六类场景，避免只优化单一模块。

## 设计策略
- `chat.message`：补齐角色、目标、输出结构、风格要求、边界条件和交付格式。
- `lyrics.values.prompt`：补齐叙事视角、情绪递进、意象、押韵/语言边界、禁忌点。
- `music.values.prompt`：补齐乐器、节奏氛围、段落推进、使用场景和制作方向。
- `cover.values.prompt`：补齐构图、镜头语言、材质、光线、留白、排版空间与禁止元素。
- `speech.values.text`：扩充为更像真实播报稿/口播稿/朗读稿的完整内容，而不是一句话样例。
- `covervoice.values.prompt`：补齐音色、咬字、呼吸、情绪、舞台感和不希望出现的演唱问题。

## 顺序化 TODO
1. 审查当前模板库条目数量、字段分布与场景类型。
2. 为模板长文本引入统一的多行拼接方式，保证源码可维护。
3. 先扩写 `chat` 模板，确保工作台默认入口质量先提升。
4. 扩写 `lyrics`、`music`、`cover` 模板，提升创作类场景首轮命中率。
5. 扩写 `speech`、`covervoice` 模板，提升声音类模板的可直接使用程度。
6. 执行语法检查和静态回归，确认模板改写未破坏前端加载。
7. 记录执行复盘、遗漏点和后续可继续演进方向。

## 验收标准
- 所有模板条目都从短句升级为明显更完整的任务说明。
- 用户点击任意模板后，输入框中出现的内容具备明确目标、输出格式和风格约束。
- 代码结构仍保持可读，不因长字符串堆叠而难以维护。
- `public/js/app-shell.js` 语法检查通过，页面静态断言无回归。
