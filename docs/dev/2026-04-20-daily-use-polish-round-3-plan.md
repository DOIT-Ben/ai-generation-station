# 2026-04-20 Daily Use Polish Round 3 Plan

## 目标
- 继续把当前版本从“已经顺手”推进到“长期天天用更省心”的状态。
- 不做后端协议扩展，优先通过前端信息密度、状态反馈和高频操作路径优化提升体验。
- 本轮重点覆盖三个高频面：聊天输入区、个人中心安全表单、后台用户管理页。

## 设计方向
- 聊天输入区：从“一个输入框 + 发送按钮”升级成更像日常主工作流的 composer。
- 个人中心：从“静态设置页”升级成更明确的安全操作页，让密码修改过程少出错。
- 管理后台：从“功能堆叠页”升级成“管理员日常操作台”，让用户搜索、概览和动作更靠近。

## 顺序化 TODO
1. 审查聊天输入区、个人中心密码表单、后台用户列表的现状与约束。
2. 升级聊天输入区：增加草稿状态、清空输入、快捷键提示和发送按钮状态同步。
3. 升级个人中心安全表单：增加实时密码校验与匹配反馈，减少提交后才报错。
4. 升级后台用户管理：增加概览指标、用户搜索和更正式的用户管理卡片样式。
5. 更新执行文档，执行语法检查与前端烟测，完成提交推送。

## 验收标准
- 聊天输入区在“空白 / 已输入 / 正在生成 / 已排队”几种状态下都给出明确反馈。
- 个人中心改密前即可看到密码是否满足要求，而不是只在提交后报错。
- 管理后台能先看到用户概况，再快速搜索和操作目标用户，不再像历史记录列表。
