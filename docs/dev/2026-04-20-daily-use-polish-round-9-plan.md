# 2026-04-20 Daily Use Polish Round 9 Plan

## 目标
- 继续推进“长期天天用”的聊天页打磨，聚焦信息层级、会话列表操作密度和输入区快捷起步体验。
- 本轮继续限定在前端聊天页，不改后端接口、不改已有会话与流式协议。

## 设计方向
- 侧栏信息层级：减少搜索反馈和会话行操作对视线的抢占，让“找到会话”和“继续当前会话”更直观。
- 主头部与输入区：让当前会话状态、快捷开始动作和输入区提示形成一条连续链路，而不是分散在多个弱关联区块里。
- 视觉收口：深浅主题、空白会话和窄屏下都要保持紧凑，不增加新的杂乱感。

## 顺序化 TODO
1. 复核当前聊天页视觉与交互痛点，补齐 round-9 文档边界与验收口径。
2. 重排会话侧栏：优化搜索反馈结构、会话行操作区和侧栏概览层级。
3. 优化聊天主区：强化当前会话头部状态表达，收紧空白会话启动区的视觉组织。
4. 优化输入区：补充快捷起步动作带与草稿态联动，提升空白输入时的启动效率。
5. 完成执行日志、回归测试、视觉基线更新与提交推送。

## 验收标准
- 侧栏不再出现一排排同权重 chips 抢占注意力，会话查找与当前会话定位更清楚。
- 空白会话时，用户能快速理解当前状态并一键开始，不需要在 starter 面板和输入框之间来回判断。
- 输入区在无草稿时提供低打扰的快捷动作，在有草稿或回复中时自动让位给主要任务。
- 深浅主题和响应式布局下，新结构不出现拥挤、遮挡或信息顺序混乱。
