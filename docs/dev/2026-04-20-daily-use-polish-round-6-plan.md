# 2026-04-20 Daily Use Polish Round 6 Plan

## 目标
- 继续推进“长期天天用”的聊天主路径打磨，重点从按钮反馈延伸到“上下文可见”和“连续追问更省一步”。
- 本轮不扩后端协议，只在现有聊天页上补更清晰的会话信息和更顺手的续聊动作。

## 设计方向
- 聊天阅读：消息不再只是内容块，还要让用户快速知道“是谁说的、什么时候说的、现在处于什么状态”。
- 会话管理：当前会话的模型、消息量、最近活跃时间和未发送草稿应在主视区直接可见，不必靠侧栏和输入框自己拼。
- 连续创作：用户在看完一轮回复后，应该能一键发起常见追问，而不是每次都重新组织一句话。

## 顺序化 TODO
1. 审查聊天阅读、会话概览和输入区续聊链路，补齐 round-6 文档。
2. 为聊天消息补元信息行，强化阅读过程中的时间感和状态感。
3. 在聊天主区头部增加当前会话概览条，展示模型、消息数、最近活跃和草稿状态。
4. 在输入区增加“继续推进当前对话”的快捷追问建议，支持一键填入。
5. 更新执行日志，执行 JS 语法检查与前端烟测，确认通过后提交并推送。

## 验收标准
- 聊天消息渲染后，用户能直接看出消息角色、时间与特殊状态，而不只看到正文。
- 当前对话的重要上下文能在主区头部快速扫到，不必来回看侧栏和底部草稿条。
- 当输入框为空且已有上下文时，用户可以从快捷建议里直接继续追问，减少一次组织语言的成本。
