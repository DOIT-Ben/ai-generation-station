# 2026-04-20 Daily Use Polish Round 10 Plan

## 目标
- 继续把聊天页打磨到更适合长期天天使用，重点减少重复入口、压缩侧栏管理成本、稳定归档与阅读状态切换。
- 本轮继续限制在前端聊天页，不触碰后端接口与 `public/js/app-shell.js` 的外部脏改动。

## 设计方向
- 侧栏管理流：把“当前会话定位 / 清空筛选 / 归档区展开收起”收敛成明确的工具区，不再把它们分散在多个反馈块里。
- 建议入口合并：把空白会话的快捷开始和有上下文会话的继续追问收敛为一个自适应建议带，减少输入区上方重复区块。
- 状态稳定性：让归档区、空白态、回复阅读态在深浅主题和窄屏里都更稳定，不出现信息顺序乱跳和按钮误显。

## 顺序化 TODO
1. 复核当前聊天页结构与 round-10 验收口径，建立计划、执行、回归文档。
2. 优化会话侧栏：增加工具区动作，加入归档区收起/展开能力，并清理冗余反馈。
3. 合并建议入口：将快捷开始和继续追问改造成单一自适应建议带，收紧空白会话起步链路。
4. 优化阅读与状态呈现：修正归档、空白态、回复阅读态和主题/窄屏样式的一致性。
5. 完成分段回归、文档复盘、提交与推送。

## 验收标准
- 侧栏高频动作可在一个稳定位置完成，筛选和归档状态清晰，不需要来回找按钮。
- 输入区上方只保留一个建议入口区块，并能根据“空白会话 / 已有上下文会话”自适应切换。
- 归档区可收起展开且状态稳定，空白态和阅读态不再出现误导性按钮或布局跳变。
- 深浅主题和响应式布局下，新结构依旧整洁，不出现遮挡、拥挤或层级失衡。
