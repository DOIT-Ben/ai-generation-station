## 执行记录
### TODO 1 复核现状
- 已查看当前聊天页最新视觉基线与代码结构，确认主要摩擦点集中在侧栏动作分散、输入区建议入口重复、归档区仍缺少状态收拢能力。
- 已确认本轮仍以“收敛和稳定”为主，不新增后端能力，不扩散到账户页、后台和模板库。

#### 单任务复盘
- 新问题：上一轮虽然优化了空白会话起步，但输入区和 starter 面板之间仍然有重复入口，长时间使用会感到区块偏多。
- 边界条件：不能为了收口而损失原有快捷行为，旧的 prompt 数据源和点击链路应继续复用。
- 遗漏补齐：需要把回归过程单独记录，避免执行日志里夹杂过多测试细节。

### TODO 2 优化会话侧栏
- 在侧栏概览卡里加入了稳定的工具区，把“定位当前会话 / 清空筛选 / 收起或展开归档”收敛到一个固定位置。
- 搜索反馈默认不再常驻；只有在存在搜索词或快捷筛选状态时才显示，降低默认空态的视觉噪音。
- 归档区补了本地持久化的展开/收起能力，减少会话多起来后侧栏被历史归档长期占满的问题。

#### 单任务复盘
- 新问题：真正高频的侧栏动作不该分散在多个小区域里，否则用户明明知道功能存在，还是要重新找按钮。
- 边界条件：归档收起只能是前端本地偏好，不能改变现有会话数据结构或后端返回格式。
- 遗漏补齐：归档切换不只影响列表显示，还会反向影响侧栏工具按钮和文案状态，因此补了同步函数统一收口。

### TODO 3 合并建议入口
- 将输入区上方原本分开的快捷开始条和继续追问条合并为一个自适应建议带。
- 建议带会根据当前状态在“空白会话起步”和“已有上下文继续推进”之间自动切换，不再在输入区上方同时出现两个同类入口。
- 继续复用原有 prompt 数据源与点击行为，避免新旧入口语义不一致。

#### 单任务复盘
- 新问题：上一轮的问题不是缺入口，而是入口太多，尤其在空白会话里容易形成“starter 面板 + quickstart 带 + follow-up 带”三层重复。
- 边界条件：starter 面板仍然保留在消息区作为更完整的起步面；输入区只负责更轻量的短路径建议。
- 遗漏补齐：建议带在有草稿、正在流式回复时必须自动隐藏，否则会和主任务争夺注意力。

### TODO 4 优化阅读与状态呈现
- 空白会话、筛选状态、归档折叠状态现在会在各自对应区域给出更明确的状态表达，不再通过多个弱提示拼凑。
- 深浅主题下补齐了建议带、侧栏工具按钮、归档折叠按钮的样式一致性。
- 响应式下补齐了侧栏工具区和建议带的换行规则，避免在窄屏下再次拥挤。

#### 单任务复盘
- 新问题：状态收口不仅是文案问题，更多是“默认情况下应该少显示什么”，否则长期使用仍然会觉得乱。
- 边界条件：本轮只做聊天页前端层级整理，不动消息渲染协议和后端归档逻辑。
- 遗漏补齐：页面静态断言同步增加了归档折叠和统一建议带结构，避免未来有人无意把这两个收口点删掉。
