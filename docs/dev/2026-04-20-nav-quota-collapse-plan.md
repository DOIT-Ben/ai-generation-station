# 2026-04-20 Nav Quota Collapse Plan

## 目标
- 为导航栏额度展示区增加“展开 / 收起”能力，减少侧边导航的视觉占用。
- 保留现有额度数据和刷新能力，不重做后端接口，只改前端壳层与交互。
- 收起和展开都要有顺滑动画，且用户偏好需要被记住。

## 设计策略
- 固定额度卡外壳，不再用整块 `innerHTML` 覆盖整个卡片。
- 卡片头部展示标题、摘要和切换按钮；卡片主体展示详细额度列表和刷新按钮。
- 收起状态下只保留摘要，不显示详细进度条列表。
- 折叠状态写入本地存储，避免用户每次刷新页面都重新展开。

## 顺序化 TODO
1. 审查额度卡的 HTML、样式和数据加载逻辑。
2. 改造额度卡 DOM 结构，增加头部摘要与切换按钮。
3. 为额度卡增加展开 / 收起动画与本地状态记忆。
4. 调整移动端和窄侧栏样式，避免折叠按钮挤压布局。
5. 执行语法检查与前端烟测，记录结果。

## 验收标准
- 用户可以通过按钮收起和展开额度展示。
- 折叠 / 展开过程有明显但克制的平滑动画。
- 收起后导航栏占用显著减少，但仍能看到额度摘要。
- 刷新额度后不会丢失当前折叠状态。
