# 2026-04-25 Chat Scroll Latest Button UX Plan

## 目标
- 将“回到最新回复”按钮再上提一点，并做成更明确、更好理解的浮动引导按钮。

## 范围
- `public\index.html`
- `public\js\app.js`
- `public\css\style.css`
- 与按钮样式和结构直接相关的最小测试

## 假设
- 用户当前主要不满意的是按钮离输入区太近，以及按钮缺少更明确的视觉引导。
- 不改按钮出现时机与点击行为，只改结构和 UX 表达。

## 风险
- 若按钮视觉权重过高，可能会压过聊天正文。
- 若结构调整与 JS 文案更新不同步，可能导致按钮内容渲染异常。

## TODO
1. 调整按钮结构，补充更明确的图标与文本布局。
2. 将按钮再上移一点，并优化 hover / active 手感。
3. 更新最小测试与回归验证。
4. 回写执行结果与复盘。

## 完成标准
- 按钮相对输入区更高一点，不显得压底。
- 按钮一眼能看出“回到底部 / 回到最新”的动作含义。
- 按钮在三种主题下都清晰、完整、稳定。

## 验证方式
- 目标样式测试。
- 无 live 主回归。
- 浏览器视觉回归。

## 复盘
- 新问题：未发现新的功能级问题。
- 边界条件：浮动按钮需要保持“有引导感但不过分抢戏”，因此图标尺寸和阴影只做轻量增强，没有做持续动画。
- 遗漏点：结构升级后，按钮文案更新逻辑必须针对 label 节点写入，避免把图标节点覆盖掉。
- 规划回写：本轮按钮 UX 目标已完成，无需继续扩到聊天其它浮动控件。
