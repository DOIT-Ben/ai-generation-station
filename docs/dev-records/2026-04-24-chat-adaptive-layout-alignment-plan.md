# AI 对话页按钮对齐与自适应布局优化计划

## 目标
- 将左侧会话区 `管理` 按钮高度与 `新建对话` 按钮对齐。
- 消除 AI 对话页底部明显空白，提升聊天区域有效利用率。
- 补强窗口非最大化、窗口缩放、不同分辨率与系统缩放比例下的自适应表现。

## 范围
- 仅调整 `AI 对话` 页面的前端结构、样式和必要的前端布局逻辑。
- 修改范围限定在：
  - `public\index.html`
  - `public\css\style.css`
  - `public\js\app.js`（仅当需要补布局同步逻辑时）
- 不修改后端接口、不调整聊天业务逻辑、不改其他功能页主结构。

## 假设
- 用户要的是现有聊天页的布局修正，而不是重新设计聊天页。
- “对齐”指视觉高度、垂直节奏、按钮基线与间距统一，而不是必须完全同宽。
- “利用下面空白”优先通过聊天容器高度分配与内部伸展解决，而不是新增内容块。

## 风险
- 若只拉高固定高度，可能在小窗口或高缩放下造成溢出和双滚动。
- 若只调按钮高度而不统一容器布局，窗口变化后仍可能错位。
- 若为了填满空间过度压缩输入区或消息区，可能影响聊天可用性。

## 方案比较
- 方案 A：只微调按钮高度和 margin。
  - 优点：改动最小。
  - 风险：不能真正解决底部空白和窗口变化后的布局问题。
- 方案 B：统一左侧头部按钮样式，并把聊天区改为基于可用视口高度的弹性布局。
  - 优点：能同时解决按钮对齐、底部空白和大部分窗口自适应问题。
  - 风险：需要同时调整聊天外层和内部滚动容器。
- 方案 C：引入更多 JS 动态测量窗口高度。
  - 优点：控制力更强。
  - 风险：复杂度更高，超出本轮“最小方案”。

## 采用方案
- 采用方案 B。
- 理由：在不扩大改动面的前提下，方案 B 最符合这轮“布局修正 + 自适应补强”的目标。

## 复盘
- 这轮问题本质上不是单个按钮样式错了，而是聊天页的高度链路和断点策略偏静态，导致按钮错位与底部空白一起出现。
- 最有效的做法不是继续堆固定高度，而是把聊天页恢复成“父层给高度、内部自己分配滚动”的结构。
