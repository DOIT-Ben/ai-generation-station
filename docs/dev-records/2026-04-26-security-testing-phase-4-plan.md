## 假设
- `test-security-gateway.js` 已具备临时服务、登录、注册、CSRF 引导、Cookie 组装和管理员查询能力。
- 公开注册频率限制可以通过临时服务环境变量降阈值稳定验证。
- token 失效边界可以通过预览链接与临时超短 TTL 组合验证。

## 风险
- 注册与 token 流程涉及登录态切换、预览链接与多次消费，测试顺序稍乱就容易出现假失败。
- 若 token 失效边界测试直接依赖真实时间，等待策略过短可能导致不稳定。
- 如果打出缺陷，修复要避免影响当前正常邀请和找回密码主链。

- 已验证至少以下边界：

  - 公开注册审计日志联动

  - 密码重置 token 缺失 / 重复消费 / 已失效边界中的至少两类
