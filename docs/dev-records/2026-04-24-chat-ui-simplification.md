# AI 对话界面精简记录

## 目标
- 精简当前 `AI 对话` 界面，减少信息层级和视觉噪音，让用户更快聚焦到“会话、消息、输入、发送”这条主线。

## 范围
- 仅处理 `AI 对话` 页面的前端结构、样式和必要交互呈现。
- 不改聊天主链路，不扩展新功能，不改其他功能页。

## 假设
- 当前聊天主链路可用，问题重点在于界面信息过多、层级过杂，而不是核心功能缺失。
- 本轮最优解是做“减法”，不是重做聊天页。

## 风险
- 若一次删太多，可能误伤已有辅助能力。
- 若只删 HTML 不校对脚本引用，可能产生前端空节点或状态异常。
- 若精简范围扩散到全站工作台，会放大改动面。
