# 2026-04-25 品牌字标、个人中心与认证中心改版计划

## 目标
完成三项相关的前台体验改版：
- 放大并拉高左上品牌字标，让 `AI / Generation / STATION` 更有识别度，不再显得扁。
- 重做 `account` 个人中心页面，让信息层级和结构更清晰，不再像信息块堆叠。
- 重做 `auth` 认证中心页面，移除冗余产品说明式文案，做成更标准、更符合当前主题风格的登录 / 注册 / 找回 / 激活 / 重置入口，并补上登录成功后的丝滑欢迎弹窗反馈。

## 范围
- `public\index.html`
- `public\account\index.html`
- `public\auth\index.html`
- `public\css\style.css`
- `public\js\auth-page.js`
- `public\js\account-page.js`
- `public\js\site-shell.js`
- 与上述页面直接相关的测试文件

## 假设
- “用户个人信息界面”按当前项目语境解释为 `\account\` 页面。
- 认证页参考站点 `https://api.suneora.com/login` 只借鉴“更标准、完整、精致的认证页气质”，不直接复制其具体结构。
- 登录成功后的欢迎反馈先作用于“登录”主流程；注册、激活、重置可继续保留现有成功跳转，不在本轮强行扩展。
- 当前主题体系仍保留 `dark`、`light`、`paper`，改版需兼容三种主题。

## 风险
- 认证页改版会牵连 token 激活 / 重置流程，需要保证切换逻辑不回归。
- 个人中心重排若只改视觉不改信息组织，仍可能看起来“高级但乱”。
- 登录成功欢迎弹窗涉及 toast / redirect 节奏，若处理不好会闪烁或来不及看完。
- 放大品牌字标后，门户页和工作台都要同步，否则视觉语言会断裂。

## 设计方向
- 品牌字标：
  - 保留当前三层结构，但增加整体尺寸、纵向节奏和字面高度，让品牌更像正式 lockup。
- 个人中心：
  - 从“说明卡 + 表单卡”改成“身份总览 + 安全操作 + 快捷入口”三段式结构。
  - 让用户首先看见“我是谁、当前状态如何、我能做什么”，再进入改密动作。
- 认证中心：
  - 去掉“为什么这么设计”的长段说明，改为更标准的品牌欢迎、表单分区、辅助入口。
  - 登录页要先给“欢迎回来”的情绪锚点，再给轻量说明。
  - 成功反馈改成右上角丝滑浮层，带环绕式进度条，到时自动消失并完成跳转。
