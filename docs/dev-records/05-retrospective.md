# 复盘

- 这次改动的核心不是单纯换图片，而是把品牌色从写死值切回主题语义色，否则 light / paper 主题下会持续失配。
- 现有测试对品牌结构写得比较死，后续如果还要继续调整品牌层级，最好优先断言语义类名和主题 token，而不是旧文案的精确排布。
- 登录页品牌头部比侧边栏更需要纵向节奏；同一套字体参数直接复用到表单卡片里，会更容易显得扁，后续这两类品牌区应继续分开微调。
- 当用户明确要“做成品牌”时，优先强化识别度，而不是只做保守排版修补；logo 尺寸、层级差、字距和微弱光感比单纯加字号更有效。
- 本轮安全修复暴露出一个测试维护点：当运行时代码新增外部资源白名单时，路由回归测试里的模拟 URL 也必须使用同一信任边界，否则会表现为异步任务超时而不是直观断言失败。
- 退役 UI 不应只删 DOM 入口；脚本加载、事件委托、模块断言、响应式样式和测试里的“必须存在”断言都要同步改成“不得存在”，否则很容易留下隐性死代码。
- 生成表单的可访问性修补要同时覆盖静态关联和运行时状态：只有补 `aria-describedby` 不够，空提交、输入修正、示例填充、重置和来源切换都必须同步维护 `aria-invalid` 与错误文案，否则用户会看到过期错误。
