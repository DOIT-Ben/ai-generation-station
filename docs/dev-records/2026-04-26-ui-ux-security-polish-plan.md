# 2026-04-26 UI\UX 与端口暴露风险修复计划

## 目标
- 修复用户指出的上传提示垂直对齐、登录欢迎弹窗时机、Logo 尺寸、品牌字体压扁、管理后台拥挤不美观问题。
- 使用 `ui-ux-pro-max` 的 UI\UX 方法约束现有页面美化，保持主题一致、深色\浅色\纸张主题都可读。
- 针对端口暴露风险补充本项目可落地的应用侧防护提示与部署文档，不误伤本地开发启动。
- 检查其他明显 UI\UX 问题并在规划内归类处理。

## 范围
- 前端页面与样式：`public\index.html`、`public\auth\index.html`、`public\admin\index.html`、`public\css\style.css`、`public\js\auth-page.js`、必要时 `public\js\site-shell.js`、`public\js\app.js`。
- 安全说明与配置文档：只补充端口暴露风险的运行建议与检查项，不引入新网关服务，不改线上基础设施。

## 不在范围
- 不接入真实 Nginx\Caddy\Cloudflare 部署。
- 不新增完整多页后台路由系统，除非现有单页布局无法通过分区和响应式改善。
- 不处理与本次问题无关的历史未提交改动。

## 假设
- “登录页面弹窗”指登录成功欢迎提示应在进入主页后出现，而不是在认证页等待弹窗结束后再跳转。
- “扩大 10% 整体”指侧边栏 Logo 图标视觉尺寸整体增大约 10%，同时保持导航不挤压。
- “Generation STATION 太矮”指品牌文字行高、字体拉伸和宽度限制造成视觉压缩，需要恢复更自然的字体比例和避免换行。
- 管理后台可以先通过单页内的信息架构分区、宽屏布局和卡片密度优化解决，不强制拆物理路由。

## 风险
- 现有工作区已有多处未提交改动，修改同一文件时必须只改本任务相关片段。
- 后台数据由 JS 动态渲染，CSS 改动需要兼容动态生成的用户卡片和审计日志表格。
- 登录成功后改成“先跳转再弹窗”需要跨页面暂存提示，需避免污染长期状态。
- 端口安全无法仅靠应用代码完全解决，必须明确区分应用侧可做与防火墙\网关侧必做。
