# 聊天模型最新优先排序计划

## 目标
- 将聊天模型下拉框调整为“最新模型优先”。
- 保持已接入的 GPT\o 系列模型列表不变，只优化排序与分组展示顺序。

## 范围
- 仅处理聊天模型排序与分组顺序。
- 修改范围限定在：
  - `server\routes\service.js`
  - `public\js\app.js`
- 不改聊天接口协议，不改其他功能页。

## 假设
- 用户当前要的是“最新的模型放在最上面”，而不是删减模型数量。
- 第三方模型列表接口返回顺序不等于我们要的 UI 排序，所以前端展示需要自定义排序。

## 风险
- 如果排序规则过于激进，可能让部分常用模型不在预期位置。
- 若分组顺序和模型顺序不一致，会造成视觉理解混乱。
