## 风险
- 邀请与密码重置链涉及 token 生命周期、登录态与审计查询，测试前置步骤多，若顺序错误容易得到假失败。
- 频率限制测试若不隔离临时服务配置，会被历史请求污染。
- 如果测试打出实现缺陷，修复需要谨慎避免影响正常登录与管理员流。

## 完成标准
- 已验证至少以下边界：
  - 找回密码频率限制
  - 管理员创建用户或管理员重置密码频率限制中的至少一类
  - 邀请发放 / 重发 / 撤销的审计日志完整性
  - 管理员重置密码的审计日志完整性
- 若发现问题，完成最小修复并通过回归；若未发现问题，完整记录结论与残余风险。
