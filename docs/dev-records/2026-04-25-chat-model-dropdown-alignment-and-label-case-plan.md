# 2026-04-25 Chat Model Dropdown Alignment And Label Case Plan

## 目标
- 修复聊天模型下拉框内容未全部左对齐的问题。
- 统一模型名称展示大小写，重点处理 `gpt` / `GPT` 混用导致的视觉不一致。
- 保持真实模型 ID、接口参数和缓存数据语义不变，只优化 UI 展示。

## 范围
- 仅检查并修改聊天模型下拉框相关样式、模型展示标签生成逻辑和必要测试。
- 不改动模型调用协议、不新增模型、不重构无关下拉框。
- 若发现其他下拉框也共用同一基础类导致同类对齐问题，只做不破坏现有布局的基础对齐修正，并记录原因。

## 假设
- 用户指出的“下拉框”主要是聊天输入区的模型选择下拉框。
- 模型值可以继续使用小写 API ID，例如 `gpt-5.2-2025-12-11`；界面可展示为更规整的 `GPT-5.2 2025-12-11`。
- “大小写规范”优先指界面可读标签，不要求修改服务端配置中的真实模型 ID。

## 风险
- 如果服务端返回的模型名称被其他地方直接复用，前端显示格式化可能需要避免影响缓存和选中值匹配。
- 过度格式化可能误改非 OpenAI 模型名称，因此只对已知模式做保守转换。
- 下拉框基础样式可能被多个组件复用，样式修正需避免改变无关控件宽度和交互。
