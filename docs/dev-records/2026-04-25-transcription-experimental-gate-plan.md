# 语音转文字实验入口降级修复记录

## 目标
修复审查报告中“语音转文字作为一等功能展示但实际只是占位壳子”的体验落差，将该模块明确降级为实验入口，避免用户误以为已正式转写。

## 范围
- 修改 `public\index.html` 的语音转文字导航和模块文案。
- 修改 `public\js\app.js` 的占位结果逻辑，改为实验接入计划说明。
- 新增聚焦测试覆盖实验状态标识、按钮文案和不再伪装“开始提取”。
- 不接入真实转写 API、不新增后端转写服务、不引入灰度用户分组。

## 假设
- 当前最小可接受修复是明确“实验/接入中”，而不是继续展示正式提取流程。
- 文件选择和拖拽入口可以保留，用于后续接入前的 UI 验证。
- 真正的转写 API 接入需要额外选型、鉴权、费用和后端任务设计，应另行规划。

## 风险
- 用户仍能看到该模块入口，但会明确识别为实验状态。
- 本轮不实现真实转写，功能价值仍有限；只是降低误导性。
