## 目标
- 启动当前项目的可运行服务，并确认前端页面与后端接口都能访问。

## 范围
- 仅处理当前项目本地启动与可访问性验证。
- 不修改业务代码，不扩展功能。

## 假设
- 当前项目采用单个 Node.js 服务同时提供前端静态资源与后端 API。
- `package.json` 中的 `start` / `dev` 即当前有效启动入口。

## 风险
- 若已有旧进程占用默认端口，启动可能失败。
- 若本地数据库文件权限异常，服务可能无法正常初始化。
- 若用户期望“前后端分开跑”，本轮需要先按当前仓库实际结构执行并说明。
