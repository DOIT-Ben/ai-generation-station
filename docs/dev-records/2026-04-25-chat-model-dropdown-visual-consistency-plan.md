# 聊天模型下拉框视觉一致性修复记录

## 目标
- 修复聊天模型下拉框“触发器尺寸与菜单尺寸不一致、内容显示长度不协调、不同主题下文字和颜色不清晰”的问题。

## 范围
- 仅处理聊天区模型下拉框的前端结构样式与必要验证。
- 允许修改的文件限定为：
  - `public\css\style.css`
  - `public\js\app.js`
  - 与本任务直接相关的测试文件

## 假设
- 当前反馈仍针对 `chat-model-dropdown`，不扩展到其他配置型小下拉框。
- 本轮优先解决视觉一致性、可读性和长度协调，不新增搜索、折叠等功能。
- 触发器与菜单不需要做像素级同宽锁死，但需要做到“视觉上同一组件体系”，避免明显割裂。

## 风险
- 调整 `.custom-dropdown` 的尺寸与排版可能影响聊天主区顶部浮层布局。
- 如果触发器改宽或改高过多，可能压缩聊天标题区的可用空间。
- 主题样式如果只改主文字，标签、分组标题和提示条仍可能保留低对比度残留。

2. 修复 `light` 与 `paper` 主题下模型下拉的文字、标签、分组标题、提示条、hover 和 active 态颜色，保证可读性。
3. 补充或更新样式与标记验证，覆盖尺寸契约、主题契约和基础结构。

## 完成标准
- 下拉触发器与菜单在视觉上宽度协调、密度一致、内容长度呈现自然。
- 当前模型名在触发器中可稳定显示，不出现明显拥挤或高度突兀。
- 菜单项、分组标题、推荐标签、滚动提示在 `dark`、`light`、`paper` 主题下都清晰可读。
- hover、active、当前态在不同主题下有足够对比，不依赖用户猜测。
- 改动不影响模型选择与默认值同步。

## 验证方式
- 静态检查相关 CSS 和 JS 改动，确认只落在聊天模型下拉范围。
- 运行与页面结构、样式契约相关的测试。
- 若测试不足，补充最小断言，覆盖触发器尺寸和主题化样式关键字。
