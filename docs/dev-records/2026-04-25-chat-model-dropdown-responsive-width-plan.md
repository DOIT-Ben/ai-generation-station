# 聊天模型下拉框缩短与响应式适配记录

## 目标
- 将聊天模型下拉框整体缩短约 30%，并优化其在页面动态变化、浏览器缩放和断点切换时的适配表现，保证 UI 与 UX 稳定。

## 范围
- 仅处理聊天区 `chat-model-dropdown` 相关样式与必要验证。
- 允许修改的文件限定为：
  - `public\css\style.css`
  - 与本任务直接相关的测试文件

## 假设
- 用户所说“整体可以短一点”指的是聊天主区左上角的模型下拉整体宽度，不要求改模型列表内容本身。
- “短 30%”按当前视觉基线近似处理，目标是明显缩短但仍保持可用和可读。
- 本轮不新增 JS 布局逻辑，优先使用 CSS 的 `clamp`、`min`、`max` 和断点规则完成适配。

## 风险
- 下拉框缩短后，长模型名的触发器展示空间会减少，需要继续保证截断自然。
- 如果断点策略不连贯，桌面缩放和窄屏之间可能出现突兀跳变。
- 模型下拉浮层位于聊天主区顶部，若宽度收缩或位移不当，可能影响标题区观感。
