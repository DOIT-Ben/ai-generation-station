# 聊天模型展示文案优化计划

## 目标
- 优化聊天模型下拉框中的展示文案。
- 保持模型真实 `id` 不变，只让前端显示更短、更清晰、更适合 UI。

## 范围
- 仅处理聊天模型展示层文案。
- 修改范围限定在：
  - `server\routes\service.js`
  - `public\js\app.js`
- 不改聊天请求协议，不改模型真实值，不改其他功能页。

## 假设
- 用户说“继续”，表示接受当前推荐模型 + 全部模型结构，希望继续优化可读性。
- 当前最值得优化的是展示 label，而不是继续增加模型管理功能。

## 风险
- 如果别名规则不清晰，可能让用户难以对应真实模型 id。
- 如果展示文案和真实值完全脱节，后续排查会变难。
