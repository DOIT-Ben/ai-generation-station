# 聊天输入区护眼模式与模型下拉初始化修复记录

## 目标
- 修复 `chat-input` 在护眼模式下颜色不一致的问题。
- 优化聊天模型下拉框初始化加载体验，避免页面加载后先卡一下才出现下拉内容。

## 范围
- 仅处理聊天输入区护眼模式样式与聊天模型下拉初始化逻辑。
- 允许修改的文件限定为：
  - `public\css\style.css`
  - `public\js\app.js`
  - 与本任务直接相关的测试文件

## 假设
- 用户反馈的 `chat-input` 是聊天输入框及其包裹容器在 `paper` 主题下的颜色层次问题。
- 用户反馈的“初始化加载进度条的时候会卡一下才出来下拉框内容”本质是模型下拉在异步拉取模型列表前处于空白或旧占位状态。
- 本轮优先优化“首屏可感知体验”，不引入新的接口或复杂骨架屏系统。

## 风险
- 调整 `paper` 主题输入区颜色时，如果只改 textarea 不改容器或 placeholder，仍可能出现层次割裂。
- 如果初始化逻辑改得不稳，可能影响默认模型回显或后续 change 事件同步。
- 若使用本地兜底占位，需要避免与真实模型返回后产生闪烁或双重状态。

## 完成标准
- `paper` 主题下聊天输入区容器、输入文字、占位文字颜色统一且可读。
- 页面初始化时，聊天模型下拉框在模型接口返回前也有稳定可见的占位或默认内容，不出现明显空白等待。
- 模型真实数据返回后，默认值与列表仍能正常同步，不影响下拉选择功能。
- 护眼模式切换时，输入框文字不再出现明显中间色闪动。
- 若本地已有最近一次成功模型列表，则首屏优先显示缓存列表，进一步缩短真实可用内容出现时间。
